ModCREDB documentation

ModCREDB is a searchable database of transcription-factor motif and structural-model evidence. It stores precomputed motif and model files, links them to UniProt TF accessions, and lets users inspect motifs, view available structures, and scan DNA sequences with FIMO.

What Users Can Do

  • Search TFs by UniProt accession, gene/protein annotation, family, or motif evidence.
  • Inspect PWM sequence logos and download MEME motif files.
  • View linked structural models when active PDB model files are available.
  • Scan pasted DNA sequences with FIMO using selected motifs or all motifs linked to a TF.
  • Download FIMO hit tables and binned profile summaries from DNA scans.

Important Interpretation

PWM evidence and structural-model evidence are related but not identical. Many TFs have usable JASPAR/CIS-BP or homology-transferred motifs without a local structural model. Some model records have a PDB structure but no usable PWM matrix, so they can be shown as model evidence but cannot be used for FIMO scanning.

A no-matrix motif file is a file-level state. It does not mean that the whole TF failed, and it does not necessarily mean that the structural model is absent.

Evidence Sources

Evidence What it means Used for FIMO? Structure/model?
JASPAR Curated public TF binding motifs imported from the local JASPAR archive. Yes, when the matrix is present. No local model required.
CIS-BP Public TF motif evidence, often from experimental or curated motif datasets. Yes, when the matrix is present. No local model required.
Identical / Homologous PWM Motif evidence assigned directly to the TF or transferred from a similar TF. Yes, when the local motif file is present. No local model required.
ModCRE Structure-based predicted PWM/model evidence from the precomputed ModCRE/ModCRElib-related dataset provided for this DB. Only when the MEME file contains a usable matrix. Often yes; linked model PDBs are shown when available.
AlphaFold/AF3 Predicted PWM/model evidence derived from AF3-related structural predictions in the provided dataset. Only when the MEME file contains a usable matrix. Often yes; linked model PDBs are shown when available.
FIMO scan MEME/FIMO is used to scan user-provided DNA with selected PWM matrices. This is the scan engine. It scans motifs; it does not create new models.

Advanced Data Quality Notes

This section is for data import and debugging. It is kept separate from the main snapshot because debug/intermediate folders should not be interpreted as public biological failure counts.

How To Read No-Matrix Counts

A no-matrix file means that a specific MEME/motif file has no usable PWM probability matrix. It is not a TF-level failure. A TF with one no-matrix file may still have other usable PWMs.

File-level count How many individual MEME/motif files cannot be used by FIMO.
TF-level count How many TF accessions have at least one of those files.
w=0 The MEME file explicitly reports zero matrix width. These files are kept for provenance, but not used for logos or FIMO scanning.
