# Recommended Information Architecture

## Public Navigation

- Search
- Scan DNA
- Evidence
- Documentation
- Downloads / citation information, when ready

Keep internal debug/QC pages out of the public navigation. Convert selected QC findings into curated public data-quality notes.

## TF Page

Show all evidence, not only a best annotation:

1. UniProt summary
2. Evidence overview badges by source
3. Motifs grouped by evidence type/source
4. Model/domain blocks by residue interval
5. Links to scan DNA with selected TF/domain/motif set
6. Data-quality notes for no-matrix/model-only entries

## Motif Page

1. Source/release/citation
2. Evidence interpretation
3. PWM logo and matrix status
4. FIMO-ready status
5. Linked TFs
6. Linked structure/model evidence, if any

## Model Page

1. Model source and active/debug status
2. TF and residue interval/domain
3. Template/model summary metadata
4. Actual PDB chain summary when available
5. 3D viewer and download

## Scan Page

Describe results as FIMO motif matches. Avoid labels implying structural affinity, in vivo binding, or ModCRE statistical-potential profiles unless those calculations are actually implemented.
